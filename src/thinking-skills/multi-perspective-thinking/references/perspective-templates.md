## よく使う視点のプロファイル集

### エンドユーザー視点

```
重視する価値:
- 使いやすさ
- 速度
- 信頼性
- シンプルさ

典型的な懸念:
- 複雑で使いにくい
- 遅い
- エラーが多い
- 学習コストが高い

質問すべきこと:
- これは直感的に理解できるか？
- エラー時の対処は明確か？
- 初めて使う人でも使えるか？
```

### 経営層視点

```
重視する価値:
- ROI（投資対効果）
- 戦略的整合性
- リスク管理
- 競争優位性

典型的な懸念:
- コストが高い
- 効果が不明確
- リスクが大きい
- 長期的な保守コスト

質問すべきこと:
- ビジネス価値は？
- 投資回収期間は？
- 戦略と整合しているか？
```

### 開発チーム視点

```
重視する価値:
- 保守性
- 技術的負債の回避
- 開発効率
- 技術的な面白さ

典型的な懸念:
- 技術的負債が増える
- 保守が困難
- レガシー技術
- ドキュメント不足

質問すべきこと:
- 保守しやすいか？
- テストしやすいか？
- 技術的負債は増えないか？
```

### 運用チーム視点

```
重視する価値:
- 安定性
- 監視しやすさ
- トラブルシューティングのしやすさ
- 自動化

典型的な懸念:
- 障害が起きやすい
- 原因究明が困難
- 夜間対応が増える
- 手動作業が多い

質問すべきこと:
- 障害時の対処は明確か？
- ログは十分か？
- 監視できるか？
```

### カスタマーサポート視点

```
重視する価値:
- わかりやすいエラーメッセージ
- ユーザーへの説明しやすさ
- 問題の切り分けやすさ
- FAQ作成のしやすさ

典型的な懸念:
- 説明が難しい
- 問い合わせが増える
- エラーメッセージが不親切
- 再現が困難

質問すべきこと:
- ユーザーに説明できるか？
- よくある問題の対処法は？
- サポートドキュメントは揃うか？
```
